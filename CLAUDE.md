## Running the app

```bash
cd backend && python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt && cd ..
uvicorn backend.main:app --reload --port 8001
```

Open `http://localhost:8001`. Demo login: `demo` / `demo1234` (auto-seeded on an empty database).

Or via Docker: `docker compose up --build`.

## Architecture

- **Backend** (`backend/`): FastAPI + SQLAlchemy 2.0 async ORM + SQLite (`sqlite+aiosqlite`, switchable to Postgres via `DATABASE_URL`). One router file per resource under `backend/routers/`, each mounted under `/api` in `main.py`. JWT auth (`backend/auth.py`, PyJWT + bcrypt) mirrors a standard OAuth2-password-bearer flow; `get_current_user` is a FastAPI dependency used on every protected route. `main.py`'s `lifespan` also starts an `AsyncIOScheduler` (APScheduler) that runs `push_service.check_and_notify_due_bills` once a day at `settings.BILL_REMINDER_HOUR`.
- **Frontend** (`frontend/`): no-build vanilla JS static app, served directly by FastAPI (`StaticFiles` mounted at `/` in `main.py`). `api.js` is a fetch wrapper that attaches the JWT bearer token and exposes namespaced call groups (`auth`, `transactions`, `creditCards`, `recurringBills`, `loans`, `dashboard`, `budgets`, `push`). `app.js` owns a module-level `state` object, renders pages by fully rebuilding `#pageContent.innerHTML`, and wires interactions through event delegation on `data-action`/`data-id` attributes. `sw.js` is a minimal service worker (push + notificationclick only) registered on boot; `manifest.json` + `frontend/icons/` make the app installable as a PWA.

### PIN lock is a client-side privacy screen, not encryption

`hasPinSet()`/`verifyPin()` in `app.js` hash a PIN with `crypto.subtle.digest('SHA-256', ...)` and store it in `localStorage` under `expense-pin-hash-v1`. This only gates the UI on that device/browser — clearing site storage bypasses it, and it never touches the backend. Don't confuse it with real auth; it exists so a phone left unattended doesn't show the balance at a glance.

### Important: click handlers use `.onclick =`, not `addEventListener`

`#pageContent` and `#modalHost` are persistent DOM nodes whose `innerHTML` gets rebuilt on every render — the nodes themselves are never replaced. Every render function assigns its click handler with `content.onclick = fn` (not `addEventListener`), because `addEventListener` would stack a new listener on top of every previous page's listener on repeat navigation, causing the same click to fire multiple handlers (this caused a real bug: paying a recurring bill instance created duplicate transactions). Keep using assignment-style handlers for anything bound to these two elements.

### Data model

- `Transaction` is the single ledger table for all money movement; `source` distinguishes manual entries from ones generated by paying a recurring bill (`recurring`) or recording a loan repayment (`loan_repayment`).
- `RecurringBill` (a template) + `RecurringBillInstance` (one row per `YYYY-MM` period, unique per bill+period) — `POST /api/recurring-bills/generate` creates a `pending` instance once `due_day` has passed for the current month; paying/skipping an instance is a separate user action, nothing runs automatically in the background.
- `Loan` + `LoanPayment` — money the user lent out (ลูกหนี้); `repayment_type` is `cash` | `monthly_installment` | `product_installment`. Recording a payment both inserts a `LoanPayment` and an income `Transaction`, and flips `Loan.status` to `completed` once payments cover the principal.
- All user-facing strings interpolated into `innerHTML` must go through `esc()` in `app.js` (XSS prevention) — keep this in place for any new dynamic content.

### Adding a new resource

Follow the existing router pattern: SQLAlchemy model in `models.py` → Pydantic schemas in `schemas.py` → a new file in `backend/routers/` with an `APIRouter(prefix=...)`, included in `main.py` under `/api` → matching namespaced object in `frontend/api.js` → a `render<Page>()` function in `app.js` following the same modal-form + `onclick` delegation pattern as the existing pages.
